# Content Services Tool Stack

## Selection Rule

Use one primary tool for each major job. Add another only when:

- the existing tool has a repeated limitation;
- a client requires it;
- volume justifies specialization;
- the replacement saves measurable time or money;
- the additional subscription has an approved ROI case.

Do not subscribe to multiple overlapping writing, SEO, enrichment, sending, project-management, or reporting platforms without evidence.

## Recommended Starting Stack

| Function | Primary option | Purpose |
|---|---|---|
| Operating system and SOPs | LifeOS | Canonical workflows, decisions, templates, evidence, reviews, and links |
| Project management | ClickUp or Notion | Client-facing task pipeline if LifeOS alone is not enough |
| Client files | Google Drive | Controlled client assets, drafts, and final files |
| Client AI workspace | ChatGPT Business Project or approved equivalent | Governed client context and draft production |
| SEO research | Semrush | Keywords, competitors, intent, gaps, and reporting |
| Writing and synthesis | ChatGPT Business | Briefs, drafts, revisions, repurposing, and structured outputs |
| Editing | Grammarly plus human review | Grammar, consistency, and readability |
| Design | Canva | Lead magnets, graphics, and client templates |
| PDF QA | Adobe Acrobat Pro | Accessibility, links, metadata, reading order, and file checks |
| CMS | WordPress or Webflow | Website publishing and landing pages |
| Email and CRM | Client-approved platform | Capture, tags, delivery, nurture, and lead ownership |
| Analytics | GA4 | Traffic and conversion events |
| Search reporting | Google Search Console | Queries, impressions, clicks, indexing, and page performance |
| Automation | Make, Zapier, n8n, or native integrations | Approved handoffs and reminders |
| Reporting | Looker Studio | Client-facing dashboards |
| Local account discovery | Google Maps / Places API | Identify and validate local businesses |
| B2B contacts | Apollo | Account and decision-maker discovery and enrichment |
| Enrichment orchestration | Clay | Multi-source enrichment, research, scoring, and routing |
| Professional research | LinkedIn / Sales Navigator | Role validation, context, networking, and warm outreach |

## Tool Approval Record

Before adding a paid tool, record:

- function;
- existing tool;
- repeated limitation;
- client or workflow requiring it;
- monthly and annual cost;
- setup and training time;
- expected time saved;
- expected revenue or risk reduction;
- data and permission requirements;
- replacement or cancellation plan;
- review date.

## Current Recommendation

For the first pilot:

1. Use LifeOS for governance.
2. Use ChatGPT for structured production.
3. Use Semrush or the best already-paid SEO platform—do not buy both Semrush and Ahrefs initially.
4. Use Canva and Acrobat for lead magnets.
5. Use the client's current CMS, email platform, and CRM.
6. Use Google Maps/Places for local account discovery.
7. Use Apollo for narrow contact discovery.
8. Add Clay only after the ICP, required fields, scoring rules, and small pilot are stable.
9. Use LinkedIn for validation and relationships.
10. Add a dedicated sequencer only when sending volume and deliverability operations justify it.

## Review Questions

- Which tools are actually used weekly?
- Which tools overlap?
- Which tools produce leads or save measurable time?
- Which tools store sensitive client data?
- Which tool is the source of truth for each record?
- Which subscriptions should be canceled, downgraded, or replaced?
- Is an automation stable enough to maintain?
