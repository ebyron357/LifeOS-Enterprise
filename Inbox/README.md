# Inbox

Purpose: Capture fast without organizing during capture.

## Use This For

- New ideas
- Links
- Tasks
- Screenshots notes
- Project fragments
- Business ideas
- Tool discoveries
- Meeting notes that are not processed yet

## Processing Rule

Every inbox item must become one of these:

- Project
- Business note
- Knowledge note
- SOP
- Tool note
- URL note
- Person note
- Task inside an existing project
- Archive

## Processing Checklist

For each inbox item:

- [ ] Decide the note type.
- [ ] Move it to the correct folder.
- [ ] Apply the correct template.
- [ ] Add metadata.
- [ ] Add a next action if it is a project.
- [ ] Link it to a related project, business, person, tool, or source.

## Rule

Inbox is temporary. It is not storage.
