# v1.0 Release Plan

## Overview

Version 1.0 is the completion of Phase 1 (Foundation). It represents the first production-ready release of LifeOS Enterprise — a single-operator system with projects, tasks, contacts, and authentication.

---

## Goals for v1.0

1. A working, secure, performant foundation
2. Complete API coverage of Foundation entities
3. Web application covering all Phase 1 user stories
4. Full test coverage meeting standards
5. Security review passed

---

## Scope

**In scope for v1.0:**
- Authentication (register, login, refresh, logout, password reset)
- Business units (create, switch, archive)
- Projects (full CRUD + status management + next action)
- Tasks (full CRUD + completion + inbox view)
- Contacts (full CRUD + project linking)
- Core API (OpenAPI 3.1 documented)
- Web application (responsive, accessible)

**Out of scope for v1.0:**
- Notes / Knowledge Graph (Phase 2)
- AI Agents (Phase 3)
- External Integrations (Phase 4)
- Business Modules (Phase 5)
- Automation (Phase 6)
- Multi-tenant SaaS (Phase 7)
- Mobile applications
- Team/multi-user features

## Definition of Done for v1.0

- [ ] All Phase 1 user stories have passing E2E tests
- [ ] All Phase 1 acceptance criteria are verified
- [ ] API documentation is complete and accurate (OpenAPI 3.1)
- [ ] Security review passed: no high/critical vulnerabilities
- [ ] Performance baselines met in staging environment
- [ ] Logging and monitoring operational
- [ ] Runbooks written for common operational tasks
- [ ] Disaster recovery test passed (restore from backup)

---

## Performance Targets (v1.0)

| Metric | Target |
|--------|--------|
| API P95 response (read) | ≤ 200ms |
| API P95 response (write) | ≤ 500ms |
| Web page initial load | ≤ 2s (3G) |
| Task inbox render (100 tasks) | ≤ 500ms |

---

## Security Targets (v1.0)

- [ ] SSL Labs Grade A on production domain
- [ ] OWASP ZAP scan: zero high or critical findings
- [ ] `npm audit` / `pip audit` clean
- [ ] No secrets in any committed code
- [ ] Authorization tests: cross-BU isolation verified
- [ ] Rate limiting active on all auth endpoints

---

## Post-v1.0: Phase 2 Planning

Phase 2 (Knowledge Graph) planning begins after v1.0 ships. The [PRD-002](../product/prd/PRD-002-knowledge-graph.md) spec is the starting point.
