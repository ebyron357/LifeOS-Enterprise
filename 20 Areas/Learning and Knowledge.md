# Learning and Knowledge

## Purpose

Turn selected learning material into practical knowledge that can be applied, searched, taught, reused, and delegated to AI without repeatedly reconstructing it from memory.

## Operating Standard

A learning item enters this area only when it supports a current goal, project, responsibility, business need, client need, community need, nonprofit mission, or defined personal-development objective.

Every completed learning session must produce at least one durable output:

- SOP;
- checklist;
- decision record;
- reference guide;
- template;
- course outline;
- applied project;
- quiz or flashcard set;
- NotebookLM source;
- reusable Markdown package.

## Monthly Review

- [ ] Which learning directly supported a completed action?
- [ ] Which saved material has not been applied?
- [ ] Which repeated question should become an SOP?
- [ ] Which notebooks or resources are stale?
- [ ] Which URLs are broken or no longer valuable?
- [ ] Which courses or platforms should be paused or cancelled?
- [ ] Which new capability should be learned next, and why?

## Guardrails

- Do not collect courses without an application plan.
- Do not save links without purpose and review metadata.
- Do not publish copyrighted transcripts or protected content.
- Do not treat AI summaries as verified until checked against the source.
- Do not automate an unstable workflow.
- LifeOS remains the canonical record; NotebookLM is a source-grounded analysis and study layer.
