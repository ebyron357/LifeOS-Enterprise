# Capture Processing Workflow

Purpose: Convert raw input into usable LifeOS assets without creating clutter.

## Rule

Capture quickly. Process deliberately.

## Step 1 — Capture

Put raw items in `Inbox/`.

Examples:

- Idea
- Link
- Screenshot note
- Client request
- Tool discovery
- Project task
- Meeting note
- Prompt

## Step 2 — Classify

Choose one type:

- Project
- Business
- Knowledge
- SOP
- Tool
- URL
- Person
- Task
- Archive

## Step 3 — Apply Template

Move the note to the correct folder and use the matching template.

## Step 4 — Add Metadata

Minimum required metadata:

- type
- status
- priority, if project or business
- review_date
- next_action, if project

## Step 5 — Link

Connect the note to at least one of:

- Project
- Business
- Tool
- Person
- Source
- SOP

## Step 6 — Queue Review

Every active note should have a `review_date`.

## Step 7 — Clear Inbox

The inbox is successful when it is empty or intentionally reviewed.

## Success Check

After processing, the item appears in the correct dashboard or is safely archived.
